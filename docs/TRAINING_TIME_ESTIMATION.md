# OpenUni2 训练时间估算报告
## 基于 Qwen2.5-VL-7B / LLaVA-1.5-7B + SD3.5-Medium 架构

---

## 1. 模型架构分析

### OpenUni2 (7B 版本) 组件

| 组件 | 模型 | 参数量 | 训练状态 |
|------|------|--------|---------|
| **LMM (语言-多模态模型)** | Qwen2.5-VL-7B 或 LLaVA-1.5-7B | ~7B | 冻结 (Frozen) |
| **Transformer (扩散模型)** | SD3.5-Medium | ~2.5B | 冻结 (Frozen) |
| **VAE** | AutoencoderKL | ~80M | 冻结 (Frozen) |
| **Connector** | 6层 Transformer Encoder | ~50-100M | **可训练** |
| **Projectors** | 3个线性投影层 | ~10-20M | **可训练** |
| **Meta Queries** | 可学习查询向量 | ~1M | **可训练** |

**总参数量**: ~9.6B
**可训练参数**: ~60-120M (约 1.2% 的总参数)

## 2. 训练配置假设

### 硬件配置
- **GPU**: 4x NVIDIA H100 (80GB)
- **单卡显存**: 80 GB
- **总显存**: 320 GB
- **GPU 间通信**: NVLink (900 GB/s)

### 训练超参数 (基于 OpenUni 论文)

**Pretrain 阶段:**
- Batch size per GPU: 8-16
- Global batch size: 32-64 (4 GPUs)
- Learning rate: 1e-4
- Optimizer: AdamW
- Precision: BF16 混合精度
- Gradient accumulation: 1-2 steps
- 数据集: megalith-10m + text2image-2M + cc12m (~24M 样本)

**Finetune 阶段:**
- Batch size per GPU: 4-8
- Global batch size: 16-32
- Learning rate: 5e-5
- 数据集: BLIP3o-60k (60K 样本)

## 3. 显存占用估算

### 单卡显存分配 (BF16, Batch Size=8)

```
模型权重 (冻结部分):
- Qwen2.5-VL-7B (BF16):     ~14 GB
- SD3.5-Medium (BF16):      ~5 GB
- VAE (BF16):               ~0.2 GB
小计:                        ~19.2 GB

可训练参数:
- Connector + Projectors:   ~0.2 GB (权重)
- 梯度:                     ~0.2 GB
- 优化器状态 (AdamW):       ~0.4 GB (2x参数)
小计:                        ~0.8 GB

激活值 (Batch=8, Seq=512):
- LMM 激活:                 ~8 GB
- Connector 激活:           ~2 GB
- Transformer 激活:         ~6 GB
小计:                        ~16 GB

其他开销:
- CUDA 内核/缓存:           ~2 GB
- 临时缓冲区:               ~2 GB
小计:                        ~4 GB

总计:                        ~40 GB / 80 GB
```

**结论**: 单卡 batch size=8 可行，batch size=16 可能需要 gradient checkpointing

## 4. 训练吞吐量估算

### 单步训练时间分析

**前向传播:**
- LMM (Qwen2.5-VL-7B) 推理: ~80 ms
- Connector 处理: ~10 ms
- SD3.5 Transformer 前向: ~150 ms
- VAE 编码: ~20 ms
**前向总计**: ~260 ms

**反向传播:**
- Connector 梯度: ~15 ms
- Projectors 梯度: ~5 ms
- 梯度累积和同步: ~20 ms (4 GPUs, NVLink)
**反向总计**: ~40 ms

**优化器步骤:**
- AdamW 更新: ~10 ms

**单步总时间**: ~310 ms (batch=8)

### 吞吐量计算

**单卡吞吐量**:
- 每秒样本数: 8 / 0.31 ≈ **25.8 samples/sec**

**4卡总吞吐量**:
- 每秒样本数: 25.8 × 4 ≈ **103 samples/sec**
- 每小时样本数: 103 × 3600 ≈ **370,800 samples/hour**

## 5. 训练时间估算

### Pretrain 阶段

**数据集规模**: 24M 样本 (megalith-10m + text2image-2M + cc12m)

**训练 Epochs**: 1 epoch (标准做法)

**总训练步数**:
```
Steps = 24,000,000 / 64 (global batch) = 375,000 steps
```

**训练时间**:
```
时间 = 375,000 steps × 0.31 sec/step / 4 GPUs
     = 375,000 × 0.31 / 4
     = 29,062 秒
     ≈ 8.1 小时
```

**考虑数据加载和其他开销 (×1.2)**:
```
实际时间 ≈ 8.1 × 1.2 = 9.7 小时
```

### Finetune 阶段

**数据集规模**: 60K 样本 (BLIP3o-60k)

**训练 Epochs**: 3-5 epochs (标准做法)

**总训练步数** (3 epochs):
```
Steps = 60,000 × 3 / 32 (global batch) = 5,625 steps
```

**训练时间**:
```
时间 = 5,625 steps × 0.31 sec/step / 4 GPUs
     = 5,625 × 0.31 / 4
     = 435 秒
     ≈ 0.12 小时 (7.3 分钟)
```

**考虑开销 (×1.2)**:
```
实际时间 ≈ 0.12 × 1.2 = 0.14 小时 (8.5 分钟)
```

## 8. 优化建议

### 加速训练的方法

1. **增加 Batch Size**
   - 使用 gradient checkpointing 节省显存
   - 可将 batch size 提升到 16-24 per GPU
   - 预计加速 **1.5-2x**

2. **使用 Flash Attention**
   - 在 Connector 中启用 Flash Attention 2
   - 预计加速 **1.2-1.3x**

3. **数据加载优化**
   - 使用 WebDataset 格式
   - 多进程数据加载 (num_workers=8-16)
   - 预计减少 **10-20%** 等待时间

4. **混合精度优化**
   - 使用 FP8 (H100 支持)
   - 预计加速 **1.3-1.5x**

5. **增加 GPU 数量**
   - 8x H100: 训练时间减半 (~5 小时)
   - 16x H100: 训练时间 1/4 (~2.5 小时)

### 应用所有优化后的估算

**完整配置 + 所有优化**:
- 原始时间: 10.15 小时
- 加速倍数: 2.0x (batch size) × 1.3x (flash attn) × 1.4x (FP8) ≈ 3.6x
- **优化后时间**: ~2.8 小时
